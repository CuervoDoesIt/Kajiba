    ```python
    @cli.command()

    @click.option("--dry-run", is_flag=True, help="Show what would be published without actually pushing")
    def publish(repo: Optional[str], dry_run: bool) -> None:
    ```

    Implementation follows D-04 workflow with Rich console output at each step:

    **Step 1: Resolve dataset repo.** Use `--repo` flag if provided, else read from config key `dataset_repo`, else `DEFAULT_DATASET_REPO`.

    **Step 2: Check gh auth.** Create `GitHubOps(upstream_repo=dataset_repo)`. Call `gh_ops.check_auth()`. If not success: print error "Not authenticated. Run `gh auth login` first." with `console.print("[red]Error:[/red] ...")` and `raise SystemExit(1)`. If GhResult.returncode == -1 (gh not found): print "gh CLI not found. Install from https://cli.github.com/" and `raise SystemExit(1)`.

    **Step 3: Load outbox records.** Call `_load_outbox_records()`. If empty: print "No records to publish. Submit records first with `kajiba submit`." and `raise SystemExit(1)`.

    **Step 4: Re-verify consent (D-03).** For each `(path, data)` in outbox records: validate via `validate_record(data)` to get KajibaRecord, read consent_level from `record.submission.consent_level` (default "full"), call `apply_consent_level(record, consent_level)`, then `record.model_dump(mode="json", by_alias=True)` to get the consent-verified dict. Collect into `verified_records: list[dict]`. Log warning for records that fail validation (skip them, don't abort).

    **Step 5: Get username and fork.** Call `gh_ops.get_username()` to get the GitHub username. Call `gh_ops.fork_repo()` (idempotent). If fork fails, print error and exit.

    **Step 6: Clone or update fork.** Check if `CLONE_DIR` exists and contains a `.git` directory. If yes: call `gh_ops.pull_latest(str(CLONE_DIR))`. If no: construct fork URL as `https://github.com/{username}/{repo_name}.git` (where repo_name is the last part of dataset_repo split on "/"). Call `gh_ops.clone_fork(str(CLONE_DIR), fork_url)`. If clone/pull fails, print error and exit.

    **Step 7: Create branch.** Generate branch name: `f"kajiba/publish-{datetime.now(UTC).strftime('%Y%m%d-%H%M%S')}"`. Call `gh_ops.create_branch(str(CLONE_DIR), branch_name)`.

    **Step 8: Write records.** Call `write_records_to_shards(CLONE_DIR, verified_records)`. Print count of records written.

    **Step 9: Generate catalog and README.** Call `generate_catalog(CLONE_DIR)`. Write to `CLONE_DIR / "catalog.json"` via `json.dumps(catalog, indent=2, ensure_ascii=False)`. Call `generate_readme(catalog)`. Write to `CLONE_DIR / "README.md"`.

    **Step 10: Dry run check.** If `--dry-run`: print summary of what would be published (record count, models, tiers) and exit without committing/pushing/PR.

    **Step 11: Commit, push, PR.** Call `gh_ops.commit_all(str(CLONE_DIR), f"kajiba: add {written_count} record(s)")`. Call `gh_ops.push_branch(str(CLONE_DIR), branch_name)`. Build PR title via `build_publish_pr_title(...)` and body via `build_publish_pr_body(...)`. Construct head as `f"{username}:{branch_name}"`. Call `gh_ops.create_pr(title, body, head)`. If PR creation fails but push succeeded: print "Records pushed to fork but PR creation failed. Create PR manually at https://github.com/{username}/{repo_name}" (partial success per error handling recommendation). If PR succeeds: print PR URL from stdout.

    **Rich output:** Use `console.print()` throughout with status indicators. Use `[green]` for success steps, `[yellow]` for warnings, `[red]` for errors. Show a Rich Panel at the end summarizing: records published, models, tiers, PR URL (or dry-run summary).

    **Error handling:** Each step checks GhResult.success. On failure, print descriptive error with stderr content and exit. The partial-success case (push OK, PR failed) must be handled specially — tell user to create PR manually.
