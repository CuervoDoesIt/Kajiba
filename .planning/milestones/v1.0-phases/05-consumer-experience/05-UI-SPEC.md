## Spacing Scale

Terminal spacing uses blank lines and indentation, not pixel values. Declared values follow the established codebase patterns.

| Token | Value | Usage |
|-------|-------|-------|
| inline | 2 spaces | Indented detail lines under a heading (e.g., `  Staging path: ...`) |
| paragraph | 1 blank line | Between distinct output sections (table, then panel, then hint) |
| section | 2 blank lines | Not used -- Rich Table/Panel titles provide visual separation |
| table-cell | Rich default padding | Column padding within Rich Table (no custom override) |
| panel-padding | Rich default | Panel internal padding (no custom override) |

Exceptions: None. All spacing follows existing cli.py conventions established in Phases 1-4.

---

## Typography

Terminal "typography" maps to Rich markup styles. Sizes and weights are controlled by the user's terminal, not the application. The contract declares style roles.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Body | no markup (plain text) | Default output text, table cell values |
| Label | `[bold]` | Field names in inline output (e.g., `[bold]Model:[/bold]`) |
| Column header | `style="bold"` on first column; `justify="right"` on numeric columns | Table column headers |
| Heading | `[bold]` as inline prefix | Section introductions (e.g., `[bold]Reviewing:[/bold] filename`) |
| Dim / secondary | `[dim]` | Hints, metadata source tags, missing values ("---"), footer text |

Weight rule: Only two emphasis levels -- `bold` for primary labels/headings and `dim` for secondary/hint text. Never use `[italic]` or `[underline]`.

---

## Color

All colors use Rich markup tags. The project has an established 4-color semantic palette. Phase 5 introduces no new colors.

| Role | Rich Tag | Usage |
|------|----------|-------|
| Success (primary feedback) | `[green]` | Download complete, auth check passed, operation succeeded |
| Warning (caution/empty) | `[yellow]` | Empty catalog, no filter matches, records available but none match |
| Error (failure) | `[red]` | Network error, gh CLI missing, auth failure, invalid input |
| Secondary (hints/context) | `[dim]` | Footer hints, missing metadata "---", file paths, source labels |
| Emphasis (labels/titles) | `[bold]` | Field labels, section headings, table first-column style |
| Field label (preview-style) | `[bold cyan]` | Reserved for preview command only -- NOT used in browse/download |
| Accent (progress bar) | `[bold blue]` | Progress bar task description text (per RESEARCH.md Pattern 4) |

**60/30/10 terminal equivalent:**
- 60% plain text: Table cell values, body output, JSONL content
- 30% `[dim]` + `[bold]`: Structural elements (labels, hints, column headers)
- 10% `[green]`/`[yellow]`/`[red]`: Semantic feedback only (success/warning/error)

Accent reserved for: Progress bar task description (`[bold blue]{task.description}`), download completion summary count values. Never used for table data or labels.

## Component Inventory

CLI "components" are Rich rendering functions. Each maps to a distinct visual output.

### Browse: Summary Table (D-01)

```
+-----------------------------------------------------+
|              Kajiba Dataset Catalog                  |
+----------+------+--------+--------+-------+----------+
| Model    | Gold | Silver | Bronze | Total | Avg Score|
+----------+------+--------+--------+-------+----------+
| Llama 3  |   12 |     28 |      5 |    45 |     0.78 |
| GPT-4o   |    8 |     15 |    --- |    23 |     0.82 |
+----------+------+--------+--------+-------+----------+

2 model(s), 68 record(s) total. Use --model <name> for details.
```

- Column 1 "Model": `style="bold"`
- Columns 2-5 (Gold, Silver, Bronze, Total): `justify="right"`; zero values rendered as `---`
- Column 6 "Avg Score": `justify="right"`; missing rendered as `---`
- Footer: `[dim]` markup

### Browse: Model Drill-Down (D-02)

```
+-- Model Metadata --------------------------------+
| Model: Llama 3                                   |
| Parameters: 8B, 70B                              |
| Quantization: Q4_K_M, Q8_0                       |
| Context Window: 8192, 131072                      |
+--------------------------------------------------+

+-- Tier Breakdown --------------------------------+
| Tier   | Records | Avg Score | Size              |
+--------+---------+-----------+-------------------+
| gold   |      12 |      0.91 |          2.3 MB   |
| silver |      28 |      0.74 |          5.1 MB   |
+--------+---------+-----------+-------------------+

Hardware: NVIDIA RTX 40xx (15), NVIDIA RTX 30xx (8), Apple M2 (5)
```

- Panel for metadata with `title="Model Metadata"`
- Label format: `[bold]Field:[/bold] value`
- Missing metadata: `[bold]Field:[/bold] [dim]---[/dim]`
- Tier table: first column `style="bold"`, numeric columns `justify="right"`
- Hardware line: `[dim]` markup, top 5 entries sorted by count descending
- Size format: `{N:.1f} MB` (megabytes with 1 decimal)

### Download: Progress Bar (D-07)

```
[spinner] Downloading...  [=========>          ] 45%  2.3/5.1 MB  00:12
```

- Rich Progress with columns: SpinnerColumn, TextColumn(`[bold blue]{task.description}`), BarColumn, percentage, DownloadColumn, TimeRemainingColumn
- Task total: number of shards (not bytes)
- DownloadColumn tracks cumulative bytes

### Download: Completion Summary

```
Downloaded 4 shard(s), 45 record(s), 5.1 MB.
Saved to ~/.hermes/kajiba/downloads/llama-3/gold/
```

- First line: `[green]` markup
- Second line: `[dim]` markup with resolved path

### Download: Unfiltered Confirmation (D-12)

```
This will download all 245 record(s) (42.3 MB). Continue? [y/N]:
```

- Plain text prompt via `click.confirm()` with default=False
- Only shown when no `--model` and no `--tier` filter applied

### Download: Skip-Existing Feedback

```
  Skipped (already exists): data/llama-3/gold/shard_a3.jsonl
```

- `[dim]` markup for each skipped file
- Shown inline during progress, does not advance progress bar

### Shared: No-Match Feedback (D-11)

```
No records match --model "mistral" --tier "gold".
Available models: gpt-4o, llama-3
Available tiers: gold, silver, bronze
```

- First line: `[yellow]` markup
- Available lists: plain text, comma-separated

### Shared: gh CLI Error

```
Error: gh CLI not found. Install from https://cli.github.com/
```

- `[red]Error:[/red]` prefix followed by plain text (matches existing publish/delete pattern)

## Interaction Contracts

### Browse Flow

1. Resolve dataset repo: `--repo` flag > config value > `DEFAULT_DATASET_REPO`
2. Fetch `catalog.json` via `gh api` with raw Accept header
3. On fetch failure: display error state copy (see Copywriting) and exit
4. On empty catalog: display empty state copy and exit
5. Apply filters (`--model`, `--tier`) via `filter_catalog()` pure function
6. On no matches: display no-match feedback with available options and exit
7. If `--model` specified and exactly one model matches: render drill-down view (D-02)
8. Otherwise: render summary table (D-01)

### Download Flow

1. Resolve dataset repo (same as browse)
2. Fetch `catalog.json` via `gh api`
3. On fetch failure: display error state copy and exit
4. Apply filters, collect matching shard paths
5. On no matches: display no-match feedback and exit
6. If no filters applied: show unfiltered confirmation prompt (D-12); exit on "N"
7. Create output directory: `--output` flag or `~/.hermes/kajiba/downloads/`
8. Download each shard with Rich Progress bar; skip existing files
9. Show completion summary with shard count, record count, total size
10. If `deletions.jsonl` exists in repo: download it too, show note about filtering

### Filter Behavior (D-10)

- `--model`: case-insensitive substring match against model slug and display_name
- `--tier`: exact match against tier name (Click.Choice enforced: gold, silver, bronze, review_needed)
- Multiple flags: AND composition (both must match)
- Single value per flag (no comma-separated lists)
